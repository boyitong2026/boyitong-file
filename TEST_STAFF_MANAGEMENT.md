# 员工管理修复测试清单

## 正式数据库准备

1. 在 Supabase SQL Editor 运行：
   `supabase/v2_staff_management_migration.sql`
2. 部署 Edge Function：
   `supabase functions deploy create-employee`
3. 重新部署或重启前端。

## 新增员工

- 老板进入“员工管理”
- 点击“新增员工”，弹窗应立即打开
- 表单应包含员工姓名、登录账号、初始密码、手机号和状态
- 员工姓名为空时显示具体提示
- 登录账号格式错误时显示具体提示
- 密码少于6位时显示具体提示
- 手机号格式错误时显示具体提示
- 重复登录账号应显示“登录账号已经存在”
- 保存成功后弹窗关闭，新员工立即出现在列表第一项

## 员工登录

- 退出老板账号
- 使用新员工的登录账号和初始密码登录
- 登录后应直接进入“待发货订单”
- 只能看到待发货订单和产品图片
- 可以上传销售单、发货和物流照片并完成发货
- 不应看到价格、欠款、统计、产品、客户和员工管理

## 员工状态

- 老板将员工状态改为“停用”
- 停用员工登录时应提示“该账号已停用，请联系老板”
- 老板恢复员工后，该员工应能重新登录

## 常见错误

- “Failed to send a request”：Edge Function 尚未部署或网络不可用
- “Function not found”：请部署 `create-employee`
- “服务端环境变量未配置”：检查 Supabase Edge Function 环境
- “只有老板账号可以新增员工”：当前登录账号不是有效管理员
